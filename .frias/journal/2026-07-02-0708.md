## Root cause of the original "echo back" (tasks returned input, never executed)

4 layered causes, fixed across v0.4.1–v0.5.5:
1. **`bridge.type: "noop"`** — `NoOpPiTaskBridge` returns a placeholder, never executes. → set `subprocess` on all nodes.
2. **Extension interference with `pi --print` stdout** — loaded extensions produced 0-byte stdout in non-interactive mode. → `--no-extensions` in the subprocess (opt-in).
3. **Model-router `cloud-via-a2a` cross-node loop** — the node router dispatched tasks to a placeholder model → wrong-node execution / loop. → explicit `--provider ollama --model qwen3.5:4b` in the subprocess (bypasses router).
4. **120s bridge timeout too short** for CPU-only qwen3.5:4b inference. → 300s.
