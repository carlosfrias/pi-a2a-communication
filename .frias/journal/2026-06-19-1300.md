## Key Findings

### Critical Corrections from Audit
1. **S3 path wrong**: The spec path is `/.well-known/agent-card.json` (with `.json` suffix per RFC 8615), not `/.well-known/agent.json` or `/.well-known/agent-card`
2. **S4 paths wrong**: A2A v1.0 defines `/message:send` (colon-separated HTTP/REST binding) and `/rpc` (JSON-RPC binding), not `/message/send` (slash-separated)
3. **S6 new gap**: Method names should be PascalCase (`SendMessage`, `GetTask`) per A2A v1.0 §5.3, not slash-separated (`message/send`, `tasks/get`)
4. **S1 severity downgrade**: From HIGH to MEDIUM — JSON-RPC 2.0 core spec is transport-agnostic; HTTP 200 is convention, not hard MUST
